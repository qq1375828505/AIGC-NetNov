一句话结论：这份 UI 设计稿已经被项目采用了——同一份 HTML（MD5 完全一致 b0e08baa…）被打包进了 app/src/main/assets/web/ui/完整版.html，通过 WebView + NativeBridge 混合架构加载。能用，但只覆盖了约 35% 的页面：AI 对话、会话记录、语音通话、工作区、主题 5 大块已真实对接原生后端可正常运行；而 10 个写作核心页面（作品/编辑器/资料/统计/工具/记忆/工作流/终端/智能体/设置详情）是空壳——讽刺的是，这些功能的后端能力（NovelRepository + 8 类素材 + 59 个 AI 工具 + 写作 Agent）在 Android 原生侧全部已实现且可用，只是没接进这份 HTML。

一、核心发现：一份 HTML，两套对接路径

通过 MD5 校验确认，你给的 UI 设计稿 = Android 项目里打包的那份 HTML，一个字都没改。但它在两种环境下的"后端对接方式"完全不同：

PC 浏览器（开发/调试）
  完整版.html ──fetch──▶ http://127.0.0.1:4590 ──▶ voice-call-server (Node.js)
                          （WebSocket 信令 + REST + SQLite，仅 PC mock 用）

Android App（生产）
  WebView 加载 完整版.html
       │
       ▼  WebUIContainer.kt 在 onPageFinished 注入 override 脚本
       │  把 HTML 里调 4590 的函数全部重写为调 window.NativeBridge
       │
       ▼  NativeBridge (@JavascriptInterface, 40+ 方法)
       ▼
  ChatViewModel / NovelRepository / ModelConfigManager / VoiceCallManager ...
  （Room + ObjectBox + 30+ AI Provider + 本地 llama.cpp/MNN）

WebUIContainer.kt:1074 的注释写得明明白白：

"NOTE: Local HTTP proxy (127.0.0.1:4590 / /api/sessions) is NOT wired up.
 Session history, voice calls, and chat now route through NativeBridge."

也就是说：voice-call-server（端口 4590）只是 PC 端调试用的 mock 后端，Android 手机上不跑这个服务，所有调用都被注入的 JS 脚本劫持重定向到原生。

四、Bug 与不一致问题清单

🔴 Bug 1：插件筛选函数未定义（必现崩溃）

位置：完整版.html plugin 页筛选 Tab 的 onclick="filterPluginList(...)"

问题：JS 中只定义了 switchPluginTab/filterPresetTabs，没有 filterPluginList 函数，点击筛选 Tab 会抛 ReferenceError。

影响：插件管理页筛选功能完全不可用。Android override 脚本未修复此 bug。

修复：在 HTML 中补 function filterPluginList(level){...}，或在 WebUIContainer 注入补丁。

🔴 Bug 2：包名三处不一致

位置写的包名实际 applicationId
sub-about 关于页com.operit.aificidecom.aificide.app
（app/build.gradle.kts）
sub-backup 备份路径com.aific
sub-about 存储路径/data/user/0/com.operit.aificide/files

影响：备份路径错误会导致找不到备份目录；关于页信息误导。需统一为 com.aificide.app。

🟡 Bug 3：角色关系图引用外部文件

位置：toolbox 页"角色关系图" window.open('角色关系图.html','_blank')

问题：该外部 HTML 文件在 assets 中不存在（项目原生侧有 ui/features/characterrelationship/ 完整实现，但 HTML 没接）。

修复：改为 NB.navigateToNative('characterrelationship') 跳原生页面，或补 assets 文件。

六、项目可用性结论

✅ 当前可正常运行的功能（5 大块）

  • AI 对话：真实流式输出，支持 30+ AI Provider（OpenAI/Claude/Gemini/DeepSeek/Qwen/豆包/Kimi/MiMo + 本地 llama.cpp/MNN），多 API Key 轮询、思考模式、Tool Call、多轮上下文
  • 会话记录：新建/列表/搜索/筛选/置顶/删除/导出（JSON/MD）/统计/恢复，完整 CRUD 持久化到 Room
  • 语音通话：ASR 语音识别（Sherpa/OpenAI/Deepgram/Silero VAD）→ AI 对话 → TTS 合成（OpenAI/豆包/MiniMax/MiMo/VITS），4 种模式（对话/听写/翻译/角色扮演）+ 总结注入
  • 工作区 + 文件管理：SAF 目录选择、文件 CRUD、路径安全校验
  • 主题 + 插件状态：5 套配色、插件开关持久化

⚠️ 空壳但后端已就绪的功能（接上就能用）

这 10 个页面在 HTML 端是死的，但对应的 Android 原生后端全部已实现且经过测试。要激活它们，只需做"补 NativeBridge 方法 + HTML override 重写"的对接工作，不需要新建任何后端能力：

  • 作品管理 → NovelRepository（NovelWork 实体 + CRUD）
  • 写作编辑器 → NovelEditorViewModel（章节/字数/1.5s 防抖自动保存/Markdown）
  • 资料管理 → NovelMaterialsViewModel（8 类素材：角色/设定/地点/势力/物品/伏笔/参考/待办）
  • 写作统计 → NovelStatsViewModel（字数/章节/日更/番茄预设）
  • 写作工具 → NovelToolsViewModel + NovelAgentDispatcher（6 工具 + 多轮推敲 Agent + 59 个 AI 工具）
  • 记忆库 → MemoryRepository（ObjectBox + 向量嵌入 + 自动保存 + 图谱）
  • 工作流 → WorkflowViewModel
  • 终端 → TerminalViewModel
  • 模型配置 → ModelConfigManager
  • 备份恢复 → RoomDatabaseBackupManager
