# Phase 3 实现计划

## 概述

Phase 3 是网文写作 IDE 的核心功能增强阶段，主要包括功能裁剪、资料管理重写、大纲管理、统计面板、番茄钟和角色关系图等模块。

---

## 功能清单与工作量评估

### 1. 功能裁剪物理移除

**描述**：移除 Operit 平台中与网文写作无关的功能模块，精简应用体积。

**涉及模块**：
- 移除通用聊天功能（保留网文相关的 AI 助手）
- 移除 MCP 服务器相关代码
- 移除不必要的插件系统
- 清理未使用的资源文件

**工作量评估**：
- 代码分析：2 小时
- 物理删除：4 小时
- 测试验证：2 小时
- **总计：8 小时（1 天）**

**风险**：
- 可能误删依赖代码
- 需要确保核心功能不受影响

---

### 2. 8 类资料管理页面重写

**描述**：重写资料管理页面，提升用户体验和功能完整性。

**现有实现**：
- `novel_materials_page.ts` - 已有基础实现
- 支持 8 类资料：角色、设定、地点、势力、道具、伏笔、参考资料、写作待办

**重写内容**：
- 优化 UI 布局和交互
- 添加搜索和过滤功能
- 支持批量操作
- 添加导入导出功能
- 优化表单验证

**工作量评估**：
- UI 设计：4 小时
- 前端开发：8 小时
- 后端对接：4 小时
- 测试验证：2 小时
- **总计：18 小时（2.5 天）**

**依赖**：
- 需要完成 Phase 2 的方法名修正
- 需要完成 NativeBridge 方法补全

---

### 3. 大纲管理页面

**描述**：实现小说大纲管理功能，支持大纲的创建、编辑和组织。

**功能点**：
- 大纲节点的增删改查
- 大纲层级结构管理
- 大纲与章节的关联
- 大纲模板功能
- 大纲导出功能

**现有实现**：
- `novel_outline.ui.ts` - 使用 WebView 加载 HTML 页面
- 需要重写为原生实现

**工作量评估**：
- UI 设计：4 小时
- 前端开发：12 小时
- 后端 API：6 小时
- 数据模型：2 小时
- 测试验证：3 小时
- **总计：27 小时（3.5 天）**

**依赖**：
- 需要扩展 NativeBridge 添加大纲相关方法
- 需要创建 Outline 数据模型

---

**工作量评估**：

- 后端数据：3 小时
- 测试验证：2 小时
- **总计：20 小时（2.5 天）**

**依赖**：
- 需要扩展 NativeBridge 添加更多统计方法
- 需要优化数据库查询性能

## 风险与缓解措施

### 风险 1：功能裁剪可能误删依赖代码
**缓解措施**：
- 详细分析代码依赖关系
- 使用工具进行静态分析
- 分阶段删除，逐步验证

### 风险 2：资料管理重写可能引入回归 bug
**缓解措施**：
- 保持向后兼容性
- 完善的单元测试
- 灰度发布策略

### 风险 3：大纲管理功能复杂度高
**缓解措施**：
- 分阶段实现核心功能
- 先实现基础增删改查
- 逐步添加高级功能

### 风险 4：图表库集成可能有性能问题
**缓解措施**：
- 选择轻量级图表库
- 优化数据加载
- 使用虚拟滚动
