## 二、核心决策统一（v3.0 修正版）

### 2.1 唯一存储决策

- **Room 数据库是主存储**：作品、卷、章节、8 类资料、番茄预设、番茄Agent、写作技能、统计全部存 Room。
- **公共目录 `/sdcard/Documents/OperitNovel/` 仅用于**：导出文件、备份、记忆附件/关系图数据、用户可访问的章节 `.md`。
- **导入文件**：通过 SAF 选择后直接解析入库，不复制到应用目录。
- **废弃**：任何以 `/sdcard/Operit/novel_workspace/` 为 primary storage 的方案。

### 2.2 唯一数据库版本（v3.0 修正）

> **重要修正**：经核实 `AppDatabase.kt`，Operit 当前数据库版本为 **v20**（不是 v21）。

- **基线**：Operit 当前数据库版本为 **v20**。
- **本次迁移**：`v20 → v21`。
- **新增表**：`novel_works`、`novel_volumes`、`novel_chapters`、`novel_characters`、`novel_settings`、`novel_locations`、`novel_factions`、`novel_items`、`novel_plot_hooks`、`novel_references`、`novel_todos`、`novel_setting_reminders`、`novel_custom_material_folders`、`novel_custom_material_items`、`tomato_presets`、`tomato_agents`、`novel_writing_skills`、`novel_character_relationships`、`novel_events`、`novel_event_participants`。

### 2.3 唯一 Screen 注册方式（v3.0 修正）

> **重要修正**：经核实 `ScreenRouteRegistry.kt`，Screen 注册机制如下：

**方案一：ToolPkg 插件注册（推荐，不修改原生层）**

在 ToolPkg `main.ts` 中使用：
```typescript
ToolPkg.registerNavigationEntry({
    id: "novel_works_sidebar",
    route: "toolpkg:com.operit.novelide:ui:novel_works",
    surface: "main_sidebar_plugins",  // 字符串，不是枚举
    title: { zh: "我的作品", en: "My Works" },
    icon: Icons.Book,
    order: 120,
});
ToolPkg.registerUiRoute({
    id: "novel_works",
    route: "toolpkg:com.operit.novelide:ui:novel_works",
    runtime: "compose_dsl",
    screen: novelWorksScreen,
    params: {},
    title: { zh: "我的作品", en: "My Works" },
});
```

**方案二：原生层注册（需修改 Operit 代码，用于侧边栏分组场景）**

1. 在 `NavItem.kt` 追加 NavItem。
2. 在 `OperitScreens.kt` 的 `Screen` sealed class 中追加子类。
3. `ScreenRouteRegistry` 通过**反射自动发现** `Screen` 子类（无需手动注册）。
4. 在 `ScreenRouteRegistry` 的 `hostEntryDefinitions` 列表中追加条目，指定 `surface`、`order`。

**可用的 NavigationSurface 枚举值**（经核实 `AppNavigationModels.kt:17`）：
- `MAIN_SIDEBAR_AI` — AI 助手分组
- `MAIN_SIDEBAR_TOOLS` — 工具分组
- `MAIN_SIDEBAR_PLUGINS` — 插件分组（ToolPkg 默认用这个）
- `MAIN_SIDEBAR_SYSTEM` — 系统分组
- `TOOLBOX` — 工具箱

**推荐**：网文写作 7 个入口使用方案一（ToolPkg 注册），surface 统一用 `"main_sidebar_plugins"`，order = 120~180。

### 2.6 UI 实现方式（v3.0 修正）

- **不重新写 Compose 页面**。
- **复用 `D:\工作区\项目\UI\` 的 HTML 设计稿**，通过 WebView 加载。
- **前端 → 原生通信**：HTML 中 JS 调用 `window.NativeBridge`（由原生 `@JavascriptInterface` 注入）。
- **原生 → 前端通信**：原生通过 `evaluateJavascript()` 调用 HTML 中的全局函数。
- 已对接：AI 对话、会话记录、语音通话、工作区、主题。
- 未对接（优先实现）：我的作品、写作编辑器、资料管理、大纲管理、写作统计、写作工具、智能体、模型配置、Token 用量、备份。

### 2.7 WebView + NativeBridge 机制说明

> **v3.0 新增**：明确说明前后端通信机制。

## 五、注意事项

1. **禁止兜底代码**：遵循 Operit `AGENTS.md` 规范，不写无意义的 fallback。
2. **优先 ToolPkg**：能用 ToolPkg 实现的，不修改原生层。
3. **统一存储**：所有新增业务数据必须进 Room，不允许直接写文件系统作为主存储。
4. **统一版本**：数据库迁移必须是 v20 → v21。
5. **前端以 HTML 为准**：`D:\工作区\项目\UI\` 设计稿是最终 UI，后端接口必须适配这些页面。
6. **包体积**：完成后评估 APK 大小，必要时删除 `dragonbones/`、`fbx/`、`mmd/`、`terminal/`、`web-chat/` 等模块。
7. **核实先行**：任何实现前先读 Operit 对应源码确认 API，不猜测。
