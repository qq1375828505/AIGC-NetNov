# 语音通话与自动总结技术方案

> 通用实现路径：前端 + 后端 + 第三方服务集成  
> 本方案同时覆盖「独立后端服务」与「Android 原生桥接」两种落地形态。

---

## 方案概述

本方案提供一套**可落地的技术实现路径**，帮助你在任意项目中集成「语音通话 + 通话结束后自动总结」功能。整体分为三大模块：前端交互界面、后端/原生桥接、第三方 AI 能力接入。

**核心流程：**

```
用户发起语音通话 → 实时语音流识别 → 文本送入主 AI → AI 文本回复 → TTS 语音播放
                                                             ↓
                                                   通话结束 → 完整转写生成总结 → 注入聊天界面
```

两种落地形态：

| 形态 | 适用场景 | 核心差异 |
|------|----------|----------|
| **独立后端服务** | Web、跨平台、服务端可控 | 前端通过 WebSocket 把音频流传给后端，后端调用 ASR/LLM/TTS |
| **Android 原生桥接** | Android 应用，已有语音/AI 基础设施 | 不依赖外部语音后端，直接用原生 `SpeechService` + `VoiceService` + `ChatViewModel` |

## 模块拆解与实现

### 1. 前端：通话交互界面

前端负责用户交互、音频采集与播放、实时状态展示。

| 功能点 | 技术要点 | 推荐方案 |
|--------|----------|----------|
| 音频采集 | 获取麦克风权限，采集音频流 | Web Audio API / getUserMedia（独立后端）<br>原生 `SpeechService` 直接采集（Android 桥接） |
| 音频播放 | 播放 AI 回复 | HTMLAudioElement / Web Audio API（独立后端）<br>原生 `VoiceService.speak()`（Android 桥接） |
| 实时通信 | 与后端/原生保持连接 | WebSocket（独立后端）<br>`@JavascriptInterface` + JS 回调（Android 桥接） |
| 通话状态 UI | 接通中、通话中、总结中等状态 | JS 状态管理 |
| 总结展示 | 通话结束后展示结构化总结 | Markdown 渲染组件 |

#### 独立后端：前端核心代码示例

```javascript
// 1. 获取麦克风权限
const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
const mediaRecorder = new MediaRecorder(stream);

// 2. 采集音频数据并发送
mediaRecorder.ondataavailable = (event) => {
  if (event.data.size > 0) {
    websocket.send(event.data);  // 发送音频流到后端
  }
};
mediaRecorder.start(100);  // 每 100ms 采集一帧

// 3. 接收后端消息（总结结果）
websocket.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.type === 'summary') {
    displaySummary(data.content);
  }
};
```

#### Android 桥接：前端覆盖示例

页面加载完成后，注入 `bridgeScript` 覆盖原 `127.0.0.1:4590` 调用：

```javascript
window.createBackendCall = function() {
  return Promise.resolve({ callId: 'native-call' });
};

window.openVoiceCall = async function() {
  // 请求麦克风权限
  if (NB.hasRecordAudioPermission && !NB.hasRecordAudioPermission()) {
    NB.requestRecordAudioPermission();
    // 等待用户授权...
  }
  NB.startVoiceCall('zh-CN');
  // 更新 UI：正在聆听...
};

window.endVoiceCall = function() {
  NB.stopVoiceCall();
  // 原生会自动把完整转写生成总结并注入主聊天
};

// 原生 → JS 实时回调
window._webuiOnVoiceTranscript = function(text) {
  document.getElementById('voiceSubtitle').textContent = text;
};
window._webuiOnVoiceUserMessage = function(text) {
  addVoiceMessage(text, 'user');
};
window._webuiOnVoiceAIResponse = function(text) {
  addVoiceMessage(text, 'ai');
};
```

### 2. 后端/桥接：通话服务与任务调度

#### 形态一：独立后端服务

后端是核心枢纽，负责信令转发、语音流处理、总结任务触发。

**后端职责清单：**

- **WebSocket 网关：** 管理客户端连接，转发信令（接通、挂断、状态变更）
- **语音流处理：** 接收前端音频流，按需转发给 ASR 服务
- **通话状态机：** 管理「待接通 → 通话中 → 总结中 → 已完成」状态流转
- **总结任务调度：** 通话结束后，触发语音转写 → 大模型总结流程
- **数据持久化：** 存储通话元数据、转写文本、总结结果

**后端 API 设计**

| 方法 | 接口 | 说明 |
|------|------|------|
| POST | /api/calls/start | 发起通话，返回通话 ID 和 WebSocket 连接地址 |
| POST | /api/calls/:id/end | 结束通话，触发总结任务 |
| GET | /api/calls/:id/summary | 查询通话总结结果 |
| WS | /ws/calls/:id | WebSocket 实时音频与信令通道 |

#### 形态二：Android 原生桥接

**`VoiceCallManager` 职责清单：**

- **初始化：** 通过 `SpeechServiceFactory` 和 `VoiceServiceFactory` 创建 ASR/TTS 实例
- **持续识别：** 调用 `speechService.startRecognition(continuousMode = true)` 持续监听用户语音
- **主 AI 调用：** 每识别到一句完整文本，调用 `ChatViewModel.sendTextMessage(text)`
- **TTS 播放：** 等待 AI 回复出现后，调用 `voiceService.speak(reply, interrupt = true)`
- **总结生成：** `stopCall()` 时把完整转写文本拼接成总结 prompt，再次调用 `ChatViewModel.sendTextMessage`

**`WebUIBridge` 暴露方法**

| 方法 | 功能 |
|------|------|
| `startVoiceCall(languageCode)` | 启动原生 ASR/TTS，开始持续识别 |
| `stopVoiceCall()` | 停止识别与 TTS，并触发总结 |
| `sendVoiceMessage(text)` | 在通话中手动发送文本并触发 AI 回复/TTS |
| `isVoiceCallActive()` | 查询通话是否进行中 |
| `getVoiceTranscript()` | 获取当前实时识别字幕 |
| `getVoiceCallTranscript()` | 获取完整通话转写 |
| `hasRecordAudioPermission()` | 检查是否已授予麦克风权限 |
| `requestRecordAudioPermission()` | 触发系统权限请求弹窗 |

### 3. 第三方服务：AI 能力接入

语音转写、大模型总结、语音合成均依赖第三方服务或本地引擎。

#### 语音识别 (ASR)

| 服务/引擎 | 特点 | 适用场景 |
|-----------|------|----------|
| 阿里云语音识别 | 中文准确率高，支持实时流式识别 | 国内项目首选 |
| 科大讯飞 | 语音技术领先，支持多方言 | 对识别精度要求高的场景 |
| Whisper (OpenAI) | 多语言支持好，成本低 | 多语言/国际化项目 |
| Sherpa-ncnn | 本地识别，无需联网 | 隐私/离线场景（Android 已集成） |

#### 大语言模型 (LLM)

| 服务/引擎 | 特点 | 适用场景 |
|-----------|------|----------|
| OpenAI GPT-4 | 总结能力强，API 成熟 | 海外项目/高质量总结 |
| 阿里云通义千问 | 中文优化好，国内访问稳定 | 国内项目首选 |
| Claude (Anthropic) | 长文本处理优秀 | 超长通话总结 |
| 本地 LLM（通过 ChatViewModel 配置） | 离线、可控 | 已配置本地模型的 Android 场景 |

#### 语音合成 (TTS)

| 服务/引擎 | 特点 | 适用场景 |
|-----------|------|----------|
| 阿里云语音合成 | 中文自然度高，多种音色 | AI 助手回复语音 |
| Azure TTS | 多语言支持，音色丰富 | 国际化项目 |
| OpenAI TTS | API 简单，音质好 | 海外项目 |
| MiMo / 豆包 / MiniMax | 国内厂商，集成方便 | Android 已集成 |
| VITS / Piper | 本地 ONNX 推理 | 离线/自定义音色（Android 已集成） |

#### 总结 Prompt 示例

```
请对以下语音通话内容进行总结，要求：
1. 提取核心议题和关键决策点
2. 列出双方达成的共识和待办事项
3. 标注重要时间节点和数字信息
4. 输出结构化的总结，使用 Markdown 格式

通话内容：
{{transcription}}
```

## 技术选型推荐

### 国内项目（推荐）

- **ASR：** 阿里云语音识别 / 科大讯飞
- **LLM：** 通义千问 / 文心一言
- **TTS：** 阿里云语音合成 / 豆包 / MiniMax / MiMo
- **后端：** Node.js / Python
- **前端：** React / Vue / 原生 WebView

### 海外项目

- **ASR：** Whisper API / Deepgram
- **LLM：** GPT-4 / Claude
- **TTS：** Azure TTS / ElevenLabs / OpenAI TTS
- **后端：** Node.js / Python
- **前端：** React / Vue

### 低成本/自托管

- **ASR：** Whisper 本地部署 / Sherpa-ncnn
- **LLM：** Llama / ChatGLM 本地
- **TTS：** Edge-TTS / Coqui / VITS
- **后端：** Python (FastAPI)
- **前端：** 任意框架

### Android 原生桥接（本项目）

- **ASR：** `SpeechServiceFactory` 创建，支持 Sherpa-ncnn / OpenAI STT / Deepgram
- **LLM：** 复用 `ChatViewModel` 当前配置的大模型
- **TTS：** `VoiceServiceFactory` 创建，支持 OpenAI/豆包/MiniMax/MiMo/VITS 等
- **权限：** `RECORD_AUDIO`（`AndroidManifest.xml` 声明 + 运行时请求）

## 开发步骤

### 形态一：独立后端服务

1. **搭建基础通话通道**
   实现 WebSocket 连接，完成「发起通话 → 音频采集 → 传输 → 挂断」的基础闭环，先不做总结，确保通话本身能跑通。

2. **接入语音识别 (ASR)**
   将前端采集的音频流实时转发给 ASR 服务，接收转写文本并在前端展示字幕效果，同时存储到数据库。

3. **实现总结触发机制**
   通话结束时，后端将完整转写文本发送给大模型，调用总结接口，将结果存入数据库并推送给前端。

4. **优化总结展示**
   前端设计总结展示界面，支持 Markdown 渲染、关键信息高亮、待办事项勾选等交互。

5. **接入语音合成 TTS（可选）**
   如果需要 AI 语音回复，将大模型生成的文本通过 TTS 转为语音流，回传给前端播放。

### 形态二：Android 原生桥接

1. **封装 `VoiceCallManager`**
   持有 `SpeechService` + `VoiceService`，实现 `startCall()` / `stopCall()` / `sendVoiceMessage()`。

2. **在 `WebUIBridge` 暴露语音方法**
   添加 `startVoiceCall` / `stopVoiceCall` / `sendVoiceMessage` / `hasRecordAudioPermission` 等 `@JavascriptInterface` 方法。

3. **前端注入 `bridgeScript` 覆盖语音函数**
   替换 `openVoiceCall` / `endVoiceCall` / `aiReplyWithVoice` 等函数，使其调用 `NativeBridge`。

4. **实现 Kotlin → JS 事件回调**
   通过 `onVoiceTranscriptUpdate` / `onVoiceUserMessage` / `onVoiceAIResponse` 实时更新前端字幕与消息历史。

5. **处理运行时权限**
   在 `WebUIContainer` 中注册 `ActivityResultContracts.RequestPermission()`，启动通话前检查/请求 `RECORD_AUDIO`。

## 数据库设计

### 形态一：独立后端服务

```sql
-- 通话记录表
CREATE TABLE calls (
  id VARCHAR(32) PRIMARY KEY,
  user_id VARCHAR(32) NOT NULL,
  status ENUM('pending','active','summarizing','completed') DEFAULT 'pending',
  started_at TIMESTAMP,
  ended_at TIMESTAMP,
  duration_seconds INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 转写文本表（按时间片存储）
CREATE TABLE transcriptions (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  call_id VARCHAR(32) NOT NULL,
  segment_index INT NOT NULL,
  speaker VARCHAR(16),  -- 'user' | 'agent'
  text TEXT NOT NULL,
  started_at TIMESTAMP,
  ended_at TIMESTAMP
);

-- 总结结果表
CREATE TABLE summaries (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  call_id VARCHAR(32) NOT NULL UNIQUE,
  content TEXT NOT NULL,        -- 总结正文
  key_points JSON,              -- 关键要点数组
  action_items JSON,            -- 待办事项数组
  generated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 形态二：Android 原生桥接

无需单独通话数据库表，直接复用：

- `ChatHistory` / `ChatMessage`：保存通话中的每一轮对话与最终总结
- `SharedPreferences`（可选）：保存通话临时状态
- 文件系统（可选）：保存长通话录音或完整转写文本

## 关键注意事项

### 通用

- **音频格式：** 前端采集通常为 PCM 或 WebM，需确认 ASR 服务支持的格式，必要时在后端做转码（如用 FFmpeg）
- **延迟优化：** 实时语音识别存在 200-500ms 延迟，前端可先做本地静音检测（VAD），减少无效数据上传
- **断线重连：** WebSocket 需实现心跳机制和自动重连，避免通话中意外断开
- **隐私合规：** 语音数据涉及敏感信息，需加密传输（WSS）并明确告知用户录音用途
- **成本控制：** ASR 和 LLM 按调用量计费，建议在后端做音频分片合并，减少 API 调用次数

### Android 原生桥接特有

- **权限声明：** `AndroidManifest.xml` 中必须声明 `<uses-permission android:name="android.permission.RECORD_AUDIO" />`，Android 6+ 还需运行时请求
- **生命周期：** `WebUIContainer` 离开页面或 `WebView` 销毁时，应调用 `VoiceCallManager.release()` 停止 ASR/TTS 并释放资源
- **线程模型：** `WebUIBridge` 的 `@JavascriptInterface` 方法是同步的；涉及协程的 ASR/TTS/AI 调用应放在 `VoiceCallManager` 内部管理，避免阻塞 JS 线程
- **TTS 打断：** 用户连续说话时，新的 AI 回复应调用 `voiceService.speak(..., interrupt = true)` 打断旧回复，避免语音重叠
- **当前会话绑定：** 语音通话中的对话应绑定到当前 `ChatViewModel.currentChatId`，确保消息出现在正确会话；`sendChatMessage(text, sessionId)` 可在发送前自动切换会话
- **总结注入：** 结束通话后，把完整转写作为用户 prompt 发送给主 AI，生成的总结消息会自然出现在当前聊天会话中
