## 2. WebView 注入安全

### 2.1 威胁模型

| 威胁 | 描述 | 风险等级 |
|------|------|---------|
| JS 代码注入 | 恶意脚本通过 `evaluateJavascript()` 执行 | 高 |
| `addJavascriptInterface` 暴露 | Android 4.2 以下所有方法可被反射调用 | 高（已弃用版本） |
| 文件跨域访问 | WebView 加载本地文件时跨域读取 | 中 |
| `setJavaScriptEnabled(true)` 滥用 | 启用 JS 后未限制接口暴露范围 | 中 |

### 2.2 防护措施

#### 2.2.1 NativeBridge 安全调用（已实施）

```kotlin
// ✅ 正确：Android 4.2+ 使用 @JavascriptInterface 注解
// NovelNativeBridge.kt
class NovelNativeBridge(...) {
    @JavascriptInterface  // 只有带此注解的方法才可以被 JS 调用
    fun saveChapterContent(chapterId: String, content: String, wordCount: Int): Boolean {
        // 方法实现
    }
}
```

#### 2.2.2 WebView 安全配置（建议实施）

```kotlin
// 建议的安全配置
webView.settings.apply {
    javaScriptEnabled = true  // 必须启用（网文 IDE 需要）
    // ✅ 禁用文件协议跨域访问
    allowFileAccess = false
    allowContentAccess = false
    // ✅ 禁用保存表单数据
    saveFormData = false
    // ✅ 禁用密码保存
    savePassword = false
}

// ✅ 移除系统注入的 JS 接口（Android 4.2-4.3 有漏洞）
webView.removeJavascriptInterface("searchBoxJavaBridge_")
webView.removeJavascriptInterface("accessibility")
webView.removeJavascriptInterface("accessibilityTraversal")

// ✅ 限制 API 等级
@RequiresApi(api = Build.VERSION_CODES.JELLY_BEAN_MR1)
// 在 JELLY_BEAN_MR1 以下禁用 addJavascriptInterface
```

#### 2.2.3 URL 白名单校验（建议实施）

```kotlin
// 建议：在 shouldOverrideUrlLoading 中校验 URL
webView.webViewClient = object : WebViewClient() {
    override fun shouldOverrideUrlLoading(view: WebView, url: String): Boolean {
        // ✅ 只允许加载应用内资源或白名单域名
        val allowedHosts = setOf("localhost", "127.0.0.1")
        val uri = Uri.parse(url)
        return if (uri.host in allowedHosts || url.startsWith("file:///android_asset/")) {
            false  // 允许加载
        } else {
            true   // 阻止加载
        }
    }
}
```

## 3. API 权限校验

### 3.1 威胁模型

| 威胁 | 描述 | 风险等级 |
|------|------|---------|
| API Key 泄露 | 硬编码或明文存储 API Key | 高 |
| 未授权 API 调用 | 无权限校验直接调用 AI API | 高 |
| 请求篡改 | 中间人篡改请求参数 | 中 |
| 频率限制绕过 | 无频率限制导致 API 滥用 | 中 |

### 3.2 API Key 保护（已实施部分）

```kotlin
// ✅ 正确：API Key 存储在 Room 数据库，不硬编码
// ApiKeyInfo.kt
@Entity(tableName = "api_keys")
data class ApiKeyInfo(
    @PrimaryKey val id: String,
    val key: String,        // 建议：加密存储
    val provider: String,
    val isEnabled: Boolean
)

// ⚠️ 建议：对 key 字段加密
// 使用 Android Keystore 系统加密敏感数据
```

### 3.3 API 调用权限校验（建议实施）

```kotlin
// 建议：在 Repository 层增加权限校验
class NovelRepository(private val dao: NovelDao) {
    
    suspend fun insertWork(work: NovelWork) {
        // ✅ 建议：校验调用者权限
        if (!PermissionChecker.hasWritePermission(context)) {
            throw SecurityException("无写入权限")
        }
        dao.insertWork(work)
    }
}
```

### 3.4 网络安全配置（建议实施）

```xml
<!-- res/xml/network_security_config.xml -->
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <!-- ✅ 强制 HTTPS -->
    <domain-config>
        <domain includeSubdomains="true">api.openai.com</domain>
        <domain includeSubdomains="true">api.anthropic.com</domain>
        <pin-set>
            <!-- 证书锁定（Certificate Pinning）-->
            <pin digest="SHA-256">你的证书哈希</pin>
        </pin-set>
    </domain-config>
    <!-- ✅ 禁用明文流量 -->
    <base-config cleartextTrafficPermitted="false">
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </base-config>
</network-security-config>
```

在 `AndroidManifest.xml` 中引用：

```xml
<application
    android:networkSecurityConfig="@xml/network_security_config"
    ...>
```

## 5. Android 权限最小化

### 5.1 当前权限声明（需审查）

```xml
<!-- ✅ 必要权限 -->

<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />  <!-- 番茄钟 -->

<!-- ⚠️ 审查以下权限是否必要 -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />  <!-- Android 10 以下 -->
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />  <!-- Android 10 以下 -->
<uses-permission android:name="android.permission.RECORD_AUDIO" />  <!-- TTS 输入 -->
<uses-permission android:name="android.permission.CAMERA" />  <!-- 待确认 -->

<!-- ✅ 建议：使用 Android 11+ 作用域存储，移除存储权限 -->
```

### 5.2 运行时权限校验

```kotlin
// 建议：在需要权限的操作前校验
fun checkAndRequestPermission(activity: Activity, permission: String) {
    when {
        ContextCompat.checkSelfPermission(activity, permission) == PackageManager.PERMISSION_GRANTED -> {
            // 已授权
        }
        else -> {
            // 请求权限
            ActivityCompat.requestPermissions(activity, arrayOf(permission), REQUEST_CODE)
        }
    }
}
```
