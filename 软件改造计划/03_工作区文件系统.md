## NovelRepository 核心接口

```kotlin
// data/repository/novel/NovelRepository.kt

interface NovelRepository {
    // 作品
    suspend fun createWork(title: String, genre: String = "", description: String = ""): String
    suspend fun getAllWorks(): Flow<List<NovelWork>>
    suspend fun getWork(workId: String): NovelWork?
    suspend fun updateWork(work: NovelWork)
    suspend fun deleteWork(workId: String)

    // 章节
    suspend fun createChapter(workId: String, title: String, sortOrder: Int = 0): String
    suspend fun getChapters(workId: String): Flow<List<Chapter>>
    suspend fun getChapter(chapterId: String): Chapter?
    suspend fun saveChapterContent(chapterId: String, content: String, wordCount: Int)
    suspend fun deleteChapter(chapterId: String)
    suspend fun reorderChapters(workId: String, chapterIds: List<String>)

    // 8 类资料（以角色为例）
    suspend fun createCharacter(character: NovelCharacter): String
    suspend fun getCharacters(workId: String): Flow<List<NovelCharacter>>
    suspend fun updateCharacter(character: NovelCharacter)
    suspend fun deleteCharacter(characterId: String)

    // 番茄预设
    suspend fun getTomatoPresets(): Flow<List<TomatoPreset>>
    suspend fun getTomatoPreset(presetId: String): TomatoPreset?
}
```

## 导入/导出流程

### 导入

1. 用户通过 SAF 选择文件（`NB.pickFileToImport`）。
2. ToolPkg `novel_io.ts` 调用对应解析器（TXT/Markdown/JSON/EPUB/DOCX/PDF）。
3. 解析结果直接写入 Room 数据库。
4. 原始文件不复制到应用目录。

### 导出

1. ToolPkg `novel_io.ts` 从 Room 读取作品/章节数据。
2. 生成 TXT/Markdown/JSON 内容。
3. 用户通过 SAF 选择保存位置，或保存到 `/sdcard/Documents/OperitNovel/exports/`。

## 废弃说明

以下方案不再使用：

- `/sdcard/Operit/novel_workspace/` 作为作品主目录。
- `NovelWorkspaceManager` 直接读写文件系统管理作品/章节。
- 章节内容以 `.md` 文件为主要存储形式。

以上只作为导出/备份/用户可读副本存在。
