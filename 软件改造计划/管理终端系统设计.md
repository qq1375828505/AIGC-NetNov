# AI-Fic-IDE 管理终端系统设计

## 一、Plugin / Skill / Agent 三层模型

### 一句话区别

| 层级 | 是什么 | 影响什么 | 类比 |
|------|--------|----------|------|
| **Plugin** | 功能扩展 | 软件功能 | VS Code 插件 |
| **Skill** | AI 能力包 | AI 写作风格/思维 | 给 AI 装的知识模块 |
| **Agent** | 自主 AI 角色 | 独立执行审查/创作任务 | 雇佣的专业编辑 |

### Plugin（插件）

**定义**：纯功能扩展，不改变 AI 行为，只改变软件能做什么。

| 例子 | 作用 |
|------|------|
| 语法润色 | 对选中文本做语法检查 |
| 导出 EPUB | 把小说导出为 EPUB 格式 |
| 字数统计 | 实时显示章节/全书字数 |
| 暗色主题 | 切换软件外观 |
| 语音朗读 | TTS 朗读当前章节 |

**存储结构**：
```
plugins/
  grammar-polish/
    manifest.json    # name, version, author, entry
    index.js         # 入口逻辑
    style.css        # 可选 UI
```

**生命周期**：安装 → 启用/禁用 → 卸载

---

### Skill（技能）

**定义**：AI 写作能力包，注入到 AI 的 system prompt 中，改变 AI 如何思考和输出。

| 例子 | 注入效果 |
|------|----------|
| 古风写作 | AI 输出古风文言风格 |
| 科幻世界观构建 | AI 帮你推演科技设定 |
| 对话风格模仿 | AI 模仿特定角色语气 |
| 伏笔管理 | AI 追踪和提示伏笔回收 |
| 章节节奏分析 | AI 分析每章节奏 |

**存储结构**：
```
skills/
  ancient-style/
    SKILL.md         # 核心指令（注入 prompt）
    examples.md      # 风格示例
    vocab.json       # 专用词库（可选）
```

**生命周期**：安装 → 加载（注入 prompt） → 卸载

---

### Agent（智能体）

**定义**：独立 AI 角色，有自己的角色定义，自主执行特定任务，输出审查/创作结果。

| 例子 | 做什么 |
|------|--------|
| 人物对话审查官 | 扫描所有对话，检查角色语气一致性 |
| 情节连贯性检查员 | 检查章节间逻辑漏洞 |
| 文风统一校对 | 确保全书文风一致 |
| 大纲偏离检测 | 检测正文是否偏离大纲 |

**存储结构**：
```
agents/
  dialogue-reviewer/
    AGENT.md         # 角色定义 + 任务描述
    report-template.md  # 输出模板
```

**生命周期**：安装 → 配置参数 → 运行 → 查看报告

---

### 三者关系

```
Agent 可以调用 Skill
  ↓
对话审查官 agent → 加载 "对话风格模仿" skill → 对比检查
  ↓
Agent 通过 Plugin 输出
  ↓
审查报告 → 通过 "导出" plugin 保存为 Markdown
```

**关键**：Agent 是"人"，Skill 是"知识"，Plugin 是"工具"。

## 二、终端命令系统

终端是软件的**管理中枢**，最常用的三类操作：

| 场景 | 占比 | 说明 |
|------|------|------|
| **文件管理** | 60% | 章节/人物/大纲的增删改查，日常必用 |
| **Skill/Agent** | 30% | 装技能、调智能体 |
| **Plugin/导出** | 10% | 偶尔用 |

### 完整命令列表

```
═══════════════════════════════════════════
  📁 文件管理（最常用）
═══════════════════════════════════════════

# 章节
list chapters                    # 列出所有章节（序号·名称·字数·修改时间）
create chapter "第四章·高潮"       # 新建章节（自动编号）
open chapter 4                   # 在编辑器中打开指定章节
rename chapter 4 "第四章·转折"    # 重命名章节
copy chapter 4                   # 复制章节（生成 04-高潮-副本.md）
move chapter 4 before 2          # 调整章节顺序（04 移到 02 前面）
delete chapter 4                 # 删除章节（需确认）
merge chapter 3 4                # 合并两章内容

# 人物
list characters                  # 列出所有人物（姓名·标签·关联章节数）
create character "赵铁柱"         # 新建人物档案
open character "赵铁柱"           # 打开人物详情
rename character "赵铁柱" "赵刚"   # 重命名人物
link character "赵刚" chapter 3  # 标记人物在某章出场
delete character "赵刚"          # 删除人物档案

# 大纲
list outlines                    # 列出所有大纲
create outline "支线B"            # 新建大纲
open outline "主线大纲"           # 打开大纲
rename outline "支线B" "暗线"      # 重命名大纲
delete outline "支线B"           # 删除大纲

# 通用
search "关键词"                   # 在所有文件中搜索
stats                            # 全书统计（总字数·章数·人物数·进度）
tree                             # 显示项目文件树
backup                           # 备份当前项目
open novel.yaml                  # 直接打开项目配置文件

═══════════════════════════════════════════
  🧠 Skill 管理
═══════════════════════════════════════════

skill install "古风写作"          # 安装技能
skill remove "古风写作"           # 移除技能
skill load "古风写作"             # 加载到当前 AI 会话
skill unload "古风写作"           # 卸载技能
skill list                       # 列出所有已安装技能
skill info "古风写作"             # 查看技能详情

═══════════════════════════════════════════
  🤖 Agent 管理
═══════════════════════════════════════════

agent install "对话审查官"        # 安装智能体
agent remove "对话审查官"         # 移除智能体
agent run "对话审查官"            # 运行智能体，扫描全书
agent run "对话审查官" chapter 3 # 只扫描指定章节
agent list                       # 列出所有智能体
agent report "对话审查官"         # 查看最近一次运行报告

═══════════════════════════════════════════
  🔌 Plugin 管理
═══════════════════════════════════════════

plugin install "语法润色"         # 安装插件
plugin remove "语法润色"          # 移除插件
plugin enable "语法润色"          # 启用插件
plugin disable "语法润色"         # 禁用插件
plugin list                      # 列出所有插件

═══════════════════════════════════════════
  📤 导出
═══════════════════════════════════════════

export novel --format epub       # 导出整本 EPUB
export novel --format pdf        # 导出 PDF
export novel --format txt        # 导出纯文本
export chapter 3 --format md     # 导出单章 Markdown

═══════════════════════════════════════════
  ⚙️ 系统
═══════════════════════════════════════════

status                           # 当前项目概览
help                             # 命令帮助
help file                        # 查看文件管理命令帮助
clear                            # 清屏
```

### 终端交互设计

```
┌─────────────────────────────────────────┐
│  📟 管理终端                         ✕  │
├─────────────────────────────────────────┤
│                                         │
│  > skill install "古风写作"              │
│  ✓ 正在下载...                          │
│  ✓ 安装完成: 古风写作 v1.2              │
│                                         │
│  > skill load "古风写作"                 │
│  ✓ 已加载到当前会话，AI 将使用古风风格   │
│                                         │
│  > list chapters                        │
│  1. 第一章 · 楔子 (2300字)              │
│  2. 第二章 · 初遇 (4100字)              │
│  3. 第三章 · 转折 (3800字)              │
│                                         │
│  > _                                    │
│                                         │
├─────────────────────────────────────────┤
│  [plugin 2] [skill 3] [agent 1]         │
└─────────────────────────────────────────┘
```

### 技术实现要点

- 终端底部状态栏：显示已安装数量（plugin/skill/agent）
- 命令支持 Tab 自动补全
- 支持 `↑↓` 历史命令
- 输出带颜色（成功=绿、警告=黄、错误=红）
- 长输出支持滚动

## 三、打开方式

终端的入口设计：

1. **快捷键**：`Ctrl+`` （和 VS Code 一致）
2. **底部面板**：写作区下方可拖拽弹出的终端面板
3. **主界面按钮**：工具栏放一个 `📟` 图标

**推荐**：快捷键 + 底部面板，不占主界面空间。

## 四、文件管理方案

```
项目根目录/
  novel.yaml              # 项目配置（书名、作者、章节顺序）
  chapters/
    01-楔子.md
    02-初遇.md
    03-转折.md
  characters/
    主角-林风.md
    女主-苏晚晴.md
  outlines/
    主线大纲.md
    支线大纲.md
  plugins/                # 已安装插件
  skills/                 # 已安装技能
  agents/                 # 已安装智能体
  exports/                # 导出文件
```

一切走 Markdown 文件，终端命令本质上是在操作这些文件。

## 五、优先级建议

| 优先级 | 功能 | 理由 |
|--------|------|------|
| **P0** | 文件管理全套（create/list/open/rename/move/delete/merge/copy） | 这是日常最高频操作，没有文件管理终端就没意义 |
| **P0** | search / stats / tree / backup | 项目管理必备 |
| **P0** | 终端 UI（面板 + 输入 + 输出 + 状态栏） | 先有壳 |
| P1 | `skill install/load/list` | 写作者开始需要 AI 能力定制 |
| P1 | `agent install/run/report` | 长篇小说需要自动审查 |
| P1 | `export novel` | 导出成品 |
| P2 | `plugin install/enable/disable` | 生态扩展 |
| P2 | Tab 补全、历史命令、彩色输出 | 体验优化 |
